Flagship Plugin

Spec System v0.6.0 Stable

When a task is too complex for Plan mode — when you need a reviewable plan, team collaboration before execution, and independent verification of every deliverable — the Spec System produces durable markdown specs you can share in PRs, iterate on with an AI judge, and execute with adversarial verification.

Read the Docs

How It Works

Three commands, three stages, one structured workflow.

1
Create Specs

Run /zoto-spec-create to decompose a feature into phased subtasks with dependencies, deliverables, and definitions of done.

2
Judge Quality

Run /zoto-spec-judge for an independent assessment. The judge agent reviews feasibility, risks, and completeness before execution.

3
Execute with Verification

Run /zoto-spec-execute to implement subtasks in order. Each deliverable is adversarially verified before moving on.
