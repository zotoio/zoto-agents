# Baseline Fixture Workspace

This directory is the **repo-wide baseline fixture skeleton** consumed by the eval-system runner. It is *not* a real workspace and must never be edited to reference any real-repo paths, environment variables, plugin internals, or product-specific files. Treat it as the deterministic empty-shell every per-case sandbox starts from: the eval-system stamps a copy into the host repo at `evals/fixtures/baseline/`, and the LLM runner's `prepareSandbox()` copies that staged tree into each case's tmp directory before applying the per-case `fixtures.files[]` overlays (which may overwrite any baseline file). The goal is that LLM-driven cases can issue commands like `pnpm run eval:create` inside the sandbox without crashing on missing `package.json` or absent `.cursor/`/`.zoto/eval-system/` scaffolding, while finding nothing repo-specific to leak into prompts or assertions. Files here must be deterministic — sorted JSON keys, LF line endings, no timestamps — and bare; do not add dependencies to `package.json`. Re-stamp via `pnpm run eval:baseline-stamp` after editing this template, and use the same script during subtask 14's live-repo migration.
